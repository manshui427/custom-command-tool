# CLI 契约：主命令框架 (`cct`)

**Feature**: 001-text-replace-tool | **Date**: 2026-06-01

本契约定义可执行程序 `cct` 顶层（不带子命令时）的命令行行为。对 CLI 工具而言，
命令行接口即对外契约。所有行为对应 spec 的 FR 与验收场景。

---

## 程序名

- 可执行文件：`cct`（Windows 上为 `cct.exe`）。

## 顶层用法

```text
cct [全局选项]
cct <子命令> [子命令参数...]
```

## 全局选项

| 选项 | 说明 | 对应需求 |
|------|------|----------|
| `-ls` | 列出所有已注册子命令的名称与简要描述 | FR-002 |
| `-h`, `--help` | 输出标准用法帮助（用途、用法格式、可用子命令、全局选项） | FR-003 |

## 行为契约

| 调用 | 期望输出（stdout） | 退出码 |
|------|-------------------|--------|
| `cct -ls` | 子命令清单，每行 `<别名>  <描述>`（如 `trt  文本替换工具`） | 0 |
| `cct -h` / `cct --help` | clap 风格标准帮助文本 | 0 |
| `cct`（无任何参数/子命令） | 与 `cct -ls` 完全一致 | 0 |
| `cct <未知子命令>` | 错误信息（stderr）+ 提示可用子命令 | 非 0 |

## 输出约定

- 正常结果 → stdout；错误/诊断 → stderr（CLI 标准约定，宪法原则 I）。
- `-ls` 输出示例：

```text
可用子命令：
  trt  文本替换工具
```

## 验收映射

- US3 场景 1 → `cct -ls`
- US3 场景 2 → `cct -h`
- US3 场景 3 → `cct`（无子命令等同 `-ls`）
