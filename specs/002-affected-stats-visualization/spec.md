# Feature Specification: 被影响文件统计与图形化操作界面

**Feature Branch**: `002-affected-stats-visualization`

**Created**: 2026-06-02

**Status**: Draft

**Input**: User description: "1 统计文件的时候只计算被影响的文件 2 在保持原有功能的前提下,是否可以将项目可视化"

## Clarifications

### Session 2026-06-02

- Q: 需求 2「可视化」的真实含义是什么？ → A: 把整个工具做成图形界面——将命令参数变成界面上的输入字段，点击「执行」按钮来替代手敲命令行（不是把替换结果做成树状图）。
- Q: 这个图形界面做成哪种形态？ → A: 桌面原生窗口程序（鼠标点击操作）。
- Q: 图形界面通过什么方式启动？ → A: 新增 `cct gui` 子命令启动窗口，不改变现有 `cct`(无参=列子命令)与 `cct trt ...` 行为（向后兼容）。
- Q: 点击执行后，图形界面展示多详细的结果？ → A: 汇总统计 + 被影响文件列表（含每文件替换次数，可滚动）。
- Q: 图形界面的覆盖范围？ → A: 本期为 trt 定制表单，界面架构预留扩展（呼应子命令注册表），未来子命令可接入。
- Q: 图形界面是否记住上次填写的参数？ → A: 不记住，每次打开为空表单/默认值；本期不引入本地配置存储。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 统计以"被影响的文件"为核心 (Priority: P1)

使用者在一次替换后，希望统计结果直接告诉他「有多少文件真正被改动、一共替换了多少处」，
而不必从庞大的扫描总数中自行推算受影响范围。

**Why this priority**: 当前摘要以扫描总数为主（如"已扫描 1240 个文件，修改 37 个…"），
在大型目录下"扫描 1240"会淹没真正重要的"被影响 37"。把统计聚焦到被影响的文件，
是本次最直接、独立可交付的价值，且 CLI 与图形界面都将复用它。

**Independent Test**: 在含大量文件、仅少数命中的目录上执行替换，验证统计以被影响文件数与替换次数为主体呈现，
且数字与实际改动一致。

**Acceptance Scenarios**:

1. **Given** 目录中扫描了 1240 个文件、其中 37 个发生替换，**When** 替换完成，
   **Then** 统计以"被影响 37 个文件、共替换 N 处"为核心信息呈现。
2. **Given** 没有任何文件命中替换，**When** 替换完成，**Then** 统计明确呈现"被影响 0 个文件"。
3. **Given** 用户需要完整信息，**When** 查看统计，**Then** 仍可获得扫描总数、跳过二进制数、失败数等次要信息（不丢失原有能力）。

---

### User Story 2 - 图形化操作界面替代命令行 (Priority: P2)

不熟悉命令行的使用者希望打开一个图形窗口，在表单中填写各项参数（目标目录、旧文本、新文本、规则文件、各开关），
点击「执行」按钮即可完成替换，无需记忆与手敲命令行；执行完成后在界面内看到本次的结果统计。

**Why this priority**: 图形界面大幅降低使用门槛，让非命令行用户也能使用本工具；
但核心替换能力（CLI）已经可用，图形界面是体验增强而非功能前提，故列为 P2。

**Independent Test**: 启动图形界面，在表单中填写一组替换参数并点击执行，验证其效果与等效命令行完全一致
（文件被正确替换、按设置生成备份），并在界面内显示被影响文件统计。

**Acceptance Scenarios**:

1. **Given** 用户运行 `cct gui` 启动了图形界面，**When** 在表单填入目标目录、旧文本、新文本并点击「执行」，
   **Then** 完成与 `cct trt -d <目录> -o <旧> -n <新>` 等效的替换，并在界面显示被影响文件数、总替换次数及被影响文件列表。
2. **Given** 用户在界面上切换备份、大小写、正则、撤销等开关，**When** 点击执行，
   **Then** 行为与对应命令行选项完全一致。
3. **Given** 用户填入非法输入（目录不存在、旧文本为空等），**When** 点击执行，
   **Then** 界面给出清晰的错误/校验提示，不崩溃、不产生部分破坏。
4. **Given** 用户不运行 `cct gui` 而照常使用命令行（含 `cct`、`cct -ls`、`cct trt ...`），**Then** 现有 CLI 行为与升级前完全一致（向后兼容）。
5. **Given** 一次替换涉及大目录、耗时较长，**When** 点击执行，**Then** 界面持续显示进度且不冻结，结束后展示统计与被影响文件列表。

---

### Edge Cases

- 统计必须只反映"被影响"的文件，未改动、跳过的二进制、处理失败的文件不计入核心计数。
- 撤销（undo）模式下的统计语义（被"还原"的文件如何计数与呈现）。
- 图形界面执行耗时较长（大目录）时，界面需持续反馈进度且不无响应（不冻结）。
- 图形界面缺失必填参数或参数非法时，应在执行前/执行时给出明确校验提示。
- 在无图形桌面环境（如纯无头服务器、无图形转发的远程会话）下启动图形界面：应给出明确提示而非崩溃。
- 图形界面中触发撤销等破坏性操作时，需先确认再执行。
- 执行过程中重复点击「执行」：需防止重入导致并发改动同一目录。

## Requirements *(mandatory)*

### Functional Requirements

#### 被影响文件统计（需求 1）

- **FR-001**: 替换完成后，统计 MUST 以"被影响的文件"（实际发生替换的文件）为核心进行计数与呈现。
- **FR-002**: 统计 MUST 报告被影响文件的数量与总替换次数。
- **FR-003**: 统计 MUST 仍可提供扫描总数、跳过二进制数、失败数等次要信息（保持原有能力不丢失），但不作为主体。
- **FR-004**: 当无任何文件被影响时，统计 MUST 明确呈现"被影响 0 个文件"。
- **FR-005**: 被影响文件的计数 MUST 与实际写回改动严格一致（跳过、未命中、失败的文件不计入被影响计数）。
- **FR-006**: 该统计口径 MUST 对命令行与图形界面两种使用方式同时生效，呈现一致。

#### 图形化操作界面（需求 2）

- **FR-007**: 系统 MUST 提供一个桌面图形操作界面，作为现有功能的图形前端，并通过新增的 `gui` 子命令（`cct gui`）启动；
  现有 `cct`（无参=列出子命令）与 `cct trt ...` 行为保持不变（向后兼容）。
- **FR-008**: 图形界面 MUST 将 trt 的各项参数（目标目录、旧文本、新文本、规则文件、备份、撤销、大小写、正则、进度）
  呈现为可输入或可选择的表单字段。
- **FR-009**: 图形界面 MUST 提供「执行」操作；点击后完成与等效命令行相同的替换或撤销，结果逐字节一致。
- **FR-010**: 图形界面 MUST 在执行完成后于界面内展示结果：以被影响文件为核心的汇总统计（复用需求 1），
  并附被影响文件列表（含每个文件的替换次数，列表可滚动）。
- **FR-011**: 图形界面 MUST 对非法或缺失的必填输入做出校验并给出清晰提示，不崩溃、不产生部分破坏的目录。
- **FR-012**: 图形界面 MUST 在执行期间提供进度反馈，长时间执行时界面 MUST NOT 冻结或无响应。
- **FR-013**: 图形界面 MUST 为可选的启动方式；不启动图形界面时，现有命令行的行为与输出 MUST 与升级前完全一致（向后兼容，保持原有功能）。
- **FR-014**: 图形界面 MUST 在 Windows 与 Linux 上均可运行。
- **FR-015**: 在无可用图形桌面环境时，启动图形界面 MUST 给出明确的提示信息而非崩溃。
- **FR-016**: 对撤销等破坏性操作，图形界面 MUST 在实际执行前要求用户确认。
- **FR-017**: 目标目录与规则文件 SHOULD 支持通过图形选择对话框选取，降低输入成本。
- **FR-018**: 图形界面 MUST NOT 持久化用户上次填写的参数；每次打开呈现空表单或默认值，本期不引入本地配置存储。
- **FR-019**: 图形界面的结构 MUST 便于未来接入新的子命令（呼应主命令的子命令注册表），本期实际仅实现 trt 的表单。

### Key Entities

- **被影响文件 (Affected File)**: 一次替换中实际发生了内容改动的文件，含相对目标目录的路径与替换次数。
- **影响报告 (Impact Report)**: 一次替换中全部被影响文件的集合及其汇总（被影响文件数、总替换次数），
  是命令行摘要与图形界面结果展示的共同数据来源。
- **参数表单 (Parameter Form)**: 图形界面中 trt 各参数到可输入/可选择字段的映射。
- **操作界面会话 (GUI Session)**: 一次图形界面交互，包含用户填写的参数表单、执行动作与执行结果（影响报告）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 替换后，使用者无需从扫描总数推算，即可一眼看出被影响文件数与总替换次数。
- **SC-002**: 不熟悉命令行的使用者无需查阅命令语法，即可仅通过图形界面完成一次完整的替换操作。
- **SC-003**: 图形界面执行的结果（替换、备份、撤销）与等效命令行逐字节一致。
- **SC-004**: 不使用图形界面、不使用新选项时，现有命令行行为与升级前完全一致（100% 向后兼容）。
- **SC-005**: 图形界面在 Windows 与 Linux 上均可启动并完成一次替换。
- **SC-006**: 大目录执行期间，图形界面持续响应（不出现长时间冻结/无响应），并在结束后展示统计。
- **SC-007**: 在无图形环境下启动图形界面时，给出明确提示且进程正常退出，不崩溃。

## Assumptions

- 本特性是对现有工具的增强；"保持原有功能"指默认（命令行）行为与现有版本完全一致，图形界面以可选方式提供。
- 图形界面作为**同一可执行文件**的一个新增 `gui` 子命令（`cct gui`）启动，复用现有 trt 核心逻辑，
  符合宪法"单一可执行文件 + 子命令"的定位；不引入独立的第二个程序，也不改变现有 `cct`/`cct -ls`/`cct trt` 的行为。
- 图形界面采用**桌面原生窗口**形态；本期不做浏览器网页界面、交互式 TUI。
- 本期图形界面覆盖 `trt` 子命令的全部参数；界面结构呼应主命令的子命令注册表，便于未来新增子命令接入，但本期仅实现 trt 表单。
- 图形界面**不持久化**用户上次填写的参数，每次打开为空表单/默认值；本期不引入本地配置存储。
- "被影响的文件"定义为本次替换中实际发生内容改动并被写回的文件（对应现有的 Modified 状态），
  不含未命中、跳过的二进制、以及处理失败的文件。
- 需求 1 的统计口径对命令行与图形界面同时生效；扫描/跳过/失败等次要信息以更简洁或可选（详细模式）的方式保留。
- 图形界面需要图形桌面环境运行；无图形环境时仅给出提示，命令行方式不受影响。
