# 契约变更：trt 统计输出（以被影响文件为核心）

**Feature**: 002-affected-stats-visualization | **Date**: 2026-06-02

定义需求 1 对 `trt` 结果摘要输出的契约调整。**仅调整呈现层**，替换/备份/撤销的实际结果不变（FR-009、SC-003）。

---

## 变更前（001 当前行为）

stdout 摘要以扫描总数为首，例如：

```text
已扫描 1240 个文件，修改 37 个，跳过二进制 8 个，失败 0 个，共替换 152 处。
备份已生成：../backup/backup_20260601_181530.zip
```

## 变更后（本特性）

摘要以**被影响文件**为核心呈现；扫描/跳过/失败作为次要信息保留（FR-001/FR-003）。示例：

```text
被影响 37 个文件，共替换 152 处。
（扫描 1240，跳过二进制 8，失败 0）
备份已生成：../backup/backup_20260601_181530.zip
```

> 文案为示意，实现时保持中文清晰、信息完整即可；关键是"被影响文件数 + 总替换次数"为首要、突出信息。

## 行为契约

| 场景 | 期望 |
|------|------|
| 有文件被影响 | 首要信息为"被影响 N 个文件，共替换 M 处"，N、M 与实际改动严格一致（FR-002/FR-005） |
| 无文件被影响 | 明确呈现"被影响 0 个文件"（FR-004），命令成功退出（码 0） |
| 次要信息 | 扫描总数、跳过二进制数、失败数仍可获得（FR-003），但不作为主体 |
| 实际结果 | 替换内容、备份产物、撤销效果与变更前逐字节一致（SC-003） |
| CLI 与 GUI 一致 | 两端使用同一影响报告数据，被影响文件口径一致（FR-006） |

## 对现有测试的影响

- 若现有集成测试断言了旧摘要文案（"已扫描 … 修改 …"），需更新为新口径断言。
- 替换正确性、二进制跳过、备份生成与撤销还原的断言**保持不变**（这些是行为而非文案）。
