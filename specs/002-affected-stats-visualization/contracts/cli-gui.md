# CLI 契约：`gui` 子命令

**Feature**: 002-affected-stats-visualization | **Date**: 2026-06-02

定义新增的 `cct gui` 子命令的行为契约。它启动桌面图形操作界面，是 trt 的图形前端。

---

## 用法

```text
cct gui
```

- 无必填参数；启动后所有输入在图形窗口内完成。
- 该子命令同时登记到主命令注册表，故出现在 `cct -ls` / `cct`（无参）的清单中。

## 启动行为契约

| 情形 | 行为 | 退出码 |
|------|------|--------|
| 有可用图形桌面环境 | 打开窗口，进入交互；窗口关闭后正常退出 | 0 |
| 无图形环境/窗口初始化失败 | 输出明确中文提示到 stderr，建议改用命令行 | 非 0 |
| 构建时未启用 `gui` feature | 输出"本可执行文件未包含图形界面支持"提示 | 非 0 |

对应 FR-007、FR-014、FR-015、SC-007。

## 窗口内的功能契约

1. **参数表单**（FR-008）：呈现 trt 全部参数为可输入/可选字段——
   目标目录、旧文本、新文本、规则文件、备份、撤销、大小写、正则、进度。
   目标目录与规则文件提供"选择"按钮调用系统文件/目录对话框（FR-017）。
2. **执行**（FR-009）：点击「执行」后，等效于运行对应的 `cct trt …`；替换/备份/撤销结果与命令行逐字节一致。
3. **校验**（FR-011）：执行前校验输入（目录存在、旧文本非空、规则来源齐备等），不通过则界面提示，不执行。
4. **进度**（FR-012）：执行期间显示进度（已扫描/已改动），界面不冻结；执行按钮在运行期间禁用（防重入）。
5. **结果展示**（FR-010）：执行完成后展示——
   - 汇总：被影响文件数、总替换次数（次要：扫描/跳过/失败、备份路径）；
   - 被影响文件列表：每个文件路径 + 替换次数，可滚动；列表过大时可截断并提示。
6. **撤销确认**（FR-016）：当表单为撤销操作时，点击执行后先弹确认对话框，确认后才执行。
7. **不持久化**（FR-018）：每次启动为默认/空表单，不回填历史输入。

## 向后兼容契约

- 增加 `gui` 子命令 MUST NOT 改变 `cct`（无参=列子命令）、`cct -ls`、`cct trt …` 的任何行为（FR-013、SC-004）。

## 验收映射

- US2 场景 1 → `cct gui` 启动 + 表单执行
- US2 场景 3 → 非法输入校验提示
- US2 场景 4 → 不启动 GUI 时 CLI 行为不变
- US2 场景 5 → 大目录执行界面不冻结
