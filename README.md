# custom-command-tool (`cct`)

> 自定义命令行工具：通过子命令快速完成各种自定义命令。

`cct` 是一个用 Rust 构建的单一可执行文件，以**子命令**的形式组织各项功能。
当前内置 `trt`（文本替换工具），后续可平滑扩展更多子命令。

## 功能特性

- ✅ **子命令架构** — 单一可执行文件，功能通过职责单一的子命令暴露，易于扩展。
- ✅ **流式并行处理** — 基于 `ignore` + `rayon` 边扫描边并行处理，超大目录也能秒级反馈进度。
- ✅ **智能二进制检测** — 自动跳过二进制文件（检测头部 NUL 字节），不破坏非文本数据。
- ✅ **批量替换** — 支持通过规则文件一次应用多组「旧→新」替换，采用单遍同时匹配、无级联。
- ✅ **正则与捕获组** — 支持正则模式及 `$1`/`${name}` 捕获组引用。
- ✅ **大小写控制** — 可配置大小写敏感度，对所有规则统一生效。
- ✅ **被影响文件统计** — 替换结果以"被影响文件数 + 总替换次数"为核心呈现。
- ✅ **图形操作界面** — `cct gui` 打开桌面窗口，表单填写参数、点击执行替代命令行，结果含被影响文件列表。
- ✅ **结构化备份** — 仅备份被修改的文件，汇集为单个 ZIP，内部保留原始目录层级。
- ✅ **一键撤销** — 按目标目录定位最近一次备份并还原，还原后自动清理备份。
- ✅ **崩溃安全** — 写回采用「临时文件 + 原子改名」，中途崩溃不产生半写损坏文件。
- ✅ **内存高效** — 大文件流式替换，内存占用不随文件数量或大小显著增长。
- ✅ **跨平台** — 运行于 Windows 与 Linux。

## 安装与构建

需要 Rust 稳定版工具链（1.75+）。

```bash
# 调试构建（默认含图形界面）
cargo build

# 发布构建（处理大型目录推荐，性能更佳）
cargo build --release
# 产物：target/release/cct（Windows 为 cct.exe）

# CLI-only 构建（不含图形界面，体积更小、依赖更少）
cargo build --release --no-default-features
```

> **图形界面（`gui`）特性**：默认启用，面向桌面用户。图形界面依赖 `eframe`/`egui`，
> 在 Linux 上运行/构建需要 X11 或 Wayland 等系统库。若只需命令行能力，用 `--no-default-features`
> 得到零 GUI 依赖的精简构建。

### 交叉编译 Linux 二进制（在 Windows 上）

可以在 Windows 上交叉编译出 **CLI-only** 的 Linux 静态二进制，**无需 Docker、无需安装 GCC 或 zig**：
依赖 musl 目标 + Rust 自带的 `rust-lld` 链接器。（图形界面在 Linux 需系统图形库，不参与此静态交叉编译；
如需 Linux 的 GUI 版本，请在 Linux 本机构建。）

一次性安装目标：

```bash
rustup target add x86_64-unknown-linux-musl
```

为 musl 目标指定 `rust-lld` 链接器。推荐写入**全局** cargo 配置 `~/.cargo/config.toml`
（Windows 为 `%USERPROFILE%\.cargo\config.toml`），这样所有项目都可复用，无需在每个仓库重复配置：

```toml
[target.x86_64-unknown-linux-musl]
linker = "rust-lld"
```

构建（CLI-only，排除 GUI 依赖）：

```bash
cargo build --release --no-default-features --target x86_64-unknown-linux-musl
# 产物：target/x86_64-unknown-linux-musl/release/cct
# 类型：ELF 64-bit、x86-64、全静态链接，可直接拷贝到 Linux 运行
```

musl 目标为**全静态链接**，产物不依赖目标系统的 glibc 或其他动态库，可在绝大多数 Linux 发行版上直接运行。

## 子命令：`trt`（文本替换工具）

一个高性能、跨平台的文本替换工具，专为处理大型目录（如 300GB+）和海量文件而设计。

### 基本语法

```bash
cct trt -d <目录> -o <被替换内容> -n <替换内容> [选项]
cct trt -d <目录> --rules <规则文件> [选项]
cct trt -d <目录> -u 1 [选项]
```

### 参数说明

| 参数 | 短选项 | 长选项 | 描述 | 默认值 |
|------|--------|--------|------|--------|
| directory | `-d` | `--directory` | 要处理的目录（相对或绝对路径） | 必填 |
| old_text | `-o` | `--old-text` | 要替换的文本（撤销模式可选） | — |
| new_text | `-n` | `--new-text` | 替换后的文本（撤销模式可选） | — |
| rules | | `--rules` | 多组替换规则文件（每行一组，制表符分隔） | — |
| backup | `-b` | `--backup` | 启用备份（1=启用, 0=禁用） | 1 |
| undo | `-u` | `--undo` | 撤销模式：还原上一次操作并删除备份 | 0 |
| case_sensitive | `-c` | `--case-sensitive` | 大小写敏感（1=敏感, 0=不敏感） | 1 |
| regex | `-r` | `--regex` | 使用正则表达式（1=正则, 0=字面） | 0 |
| progress | | `--progress` | 显示进度（1=显示, 0=隐藏） | 1 |

> 非撤销模式下，`-o/-n` 与 `--rules` 必须至少提供其一；同时提供时命令行单组规则会追加到规则集末尾。

### 使用示例

```bash
# 基本替换：将目录内所有文本文件的 foo 改为 bar（默认启用备份）
cct trt -d ./project -o foo -n bar

# 正则 + 捕获组：将 2026-06 形式的年月改为 06/2026
cct trt -d ./logs -r 1 -o '([0-9]{4})-([0-9]{2})' -n '${2}/${1}'

# 批量规则文件，关闭备份
cct trt -d ./project --rules ./rules.tsv -b 0

# 大小写不敏感替换
cct trt -d ./docs -o hello -n hi -c 0

# 撤销 ./project 上一次替换
cct trt -d ./project -u 1
```

### 规则文件格式

规则文件每行一组规则，旧文本与新文本以**制表符**分隔；空行与以 `#` 开头的行被忽略：

```text
# 旧文本<TAB>新文本
foo	bar
baz	qux
old-name	new-name
```

多组规则采用**单遍同时匹配**：每个位置至多被一条规则命中，重叠时按规则在文件中的先后顺序优先，
不会发生「前一条规则的结果被后一条再次替换」的级联。

## 子命令：`gui`（图形操作界面）

面向不熟悉命令行的用户，把 trt 的参数做成图形表单，点击「执行」即可完成替换，替代手敲命令行。

```bash
cct gui
```

在窗口中：

1. 填写/选择**目标目录**（可点「选择…」调用系统目录对话框）。
2. 填写**旧文本**、**新文本**，或选择**规则文件**做批量替换。
3. 按需勾选**备份 / 大小写敏感 / 正则 / 撤销**等开关。
4. 点击「**执行**」：
   - 执行期间显示进度且界面不冻结；撤销操作会先弹确认；
   - 完成后展示**被影响文件数、总替换次数**与**被影响文件列表**（每文件替换次数，可滚动）。

图形界面执行的效果与等效命令行逐字节一致。在无图形桌面环境（如纯无头服务器）运行 `cct gui`
会给出明确提示并以非零码退出，不会崩溃；此时请改用命令行 `cct trt …`。

> 图形界面需以默认特性（含 `gui`）构建；`--no-default-features` 的精简构建下 `cct gui` 会提示未包含图形界面支持。

## 备份与还原

- **备份路径**：`<目标目录同级>/backup/backup_yyyyMMddHHmmss.zip`（同一秒内多次备份会追加序号后缀以避免覆盖）。
- **层级保留**：ZIP 内部路径与相对目标目录的物理路径完全一致，可解压后直接覆盖还原。
- **来源元数据**：每个备份内含 `.cct-manifest.json`，记录来源目录，使撤销能按目录精确定位，
  多个目录共享同一 `backup/` 也互不干扰。
- **自动撤销**：`cct trt -d <目录> -u 1` 还原与该目录匹配的最近一次备份，并在成功后删除该归档。
- **手动还原**：直接将备份 ZIP 内容解压并覆盖至目标目录即可。

## 性能与实现要点

1. **并行扫描**：基于 `ignore` 的并行遍历，不等待整个目录扫描完成即开始处理已发现的文件。
2. **流式大文件**：超过阈值的大文件采用流式替换，避免整体载入内存。
3. **多模式匹配**：字面替换使用 `aho-corasick` 自动机一次扫描匹配全部规则；正则替换使用 `regex`。
4. **原子写回**：先写同目录临时文件再原子改名覆盖，保证崩溃安全。
